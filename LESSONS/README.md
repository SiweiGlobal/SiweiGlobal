# LESSONS

## 目录用途

记录错误、失败、弯路和复盘结果，说明以后如何减少同类问题再次发生。

## 适合记录的内容

- 发生了什么以及造成了什么影响
- 问题产生的主要原因
- 当时忽略的信号或错误假设
- 下次可以采取的具体改进方式

## 不适合记录的内容

- 对个人或团队的情绪化指责
- 没有事实依据的归因
- 只写失败结果、不总结改进方法的记录

## 推荐的文件命名方式

使用 `YYYY-MM-DD-复盘主题.md`，或使用 `主题-经验.md`。

## 简单示例

`2026-07-15-项目范围扩张复盘.md`：记录范围失控的原因、造成的影响和以后设置边界的方法。

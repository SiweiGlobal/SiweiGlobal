即使有一天我无法再继续，这些文字、产品、代码、图片和项目，也可以替我保存一部分经历。

## 关于 SiweiGlobal

**SiweiGlobal** 不只是一个品牌名称，也不只是一个网站或 GitHub 账号。

它更像是一个长期存在的容器。

我希望把自己做过的项目、走过的路、形成的想法、积累的经验以及对未来的探索，逐渐保存到这里。

SiweiGlobal 中的“SIWEI”，来自中国的一句话：

> 居安思危。

它提醒我，即使身处相对平静的时刻，也不要忘记变化、风险和不确定性。

世界并不会永远按照计划运行。

身体、生活、收入、行业、市场、技术和人与人之间的关系，都可能发生变化。很多事情并不会提前通知我们，也不会因为我们还没有准备好就暂停到来。

我比很多人更早地意识到这一点。

这种意识曾经让我感到不安，也让我反复思考：一个人应该怎样度过自己拥有的时间？

后来我逐渐明白，真正重要的，也许不是彻底消除所有不确定性，而是在不确定性中，仍然选择去创造一些东西。

SiweiGlobal 就是在这样的想法中产生的。

# 一、人生无法选择起点

一个人无法决定自己出生在哪里，也无法决定自己最初拥有什么条件。

我们无法选择家庭背景，无法选择身体状况，也无法选择成长过程中会遇到怎样的环境。

有些人从一开始就拥有稳定的资源、清晰的道路和足够的安全感。

有些人则需要很早面对限制、风险以及无法解释的不确定性。

我属于后者。

在成长过程中，我逐渐意识到，有些事情并不是只要足够努力就一定能够改变。

努力很重要，但努力并不能决定一切。

现实中的很多结果，还受到身体、时代、机会、资源、家庭、环境和运气的影响。

这并不是悲观，而是我对生活逐渐形成的一种理解。

一个人如果只相信“努力就一定成功”，那么当结果没有出现时，很容易把所有问题都归咎于自己。

但现实并不是一道只要认真作答就一定能够得到满分的试卷。

它更像是一条不断改变的路。

有人一开始就在平坦的地方，有人一开始就在坡道上，有人走着走着还会遇到突然出现的障碍。

我无法改变自己的起点。

但我不希望起点决定我全部的人生。

# 二、我曾经做过程序开发

我曾经接触过软件开发，也写过代码。

我使用过 Python，也理解程序、数据、系统和逻辑之间的关系。

那段经历让我第一次感受到，一段看不见、摸不着的代码，可以通过不断组合，最终变成一个能够实际运行的工具。

软件开发给我留下了一种很重要的思维方式：

> 一个复杂的问题，可以被拆成许多个具体的小问题。

当一个系统看起来非常庞大时，可以先确定它的目标，再划分模块，设计字段，建立规则，然后逐步实现。

这种思维后来也影响了我对创业、产品和人生的理解。

只是后来因为生活、工作和时间等原因，我没有一直持续进行传统的软件开发。

随着时间过去，很多曾经熟悉的代码和技术逐渐变得陌生。

有时我会觉得可惜。

曾经会做的事情，因为长期没有使用，慢慢变得生疏。曾经可以独立完成的代码，现在可能需要重新查资料、重新学习。

但我并不认为过去的经历彻底消失了。

它们仍然影响着我。

我依然习惯分析结构、建立目录、整理规则、拆分模块，也依然相信一件复杂的事情可以通过长期积累逐步完成。

后来，人工智能工具出现了。

Codex、Claude Code、本地语言模型以及各种 AI 辅助开发工具，让我重新看到了回到软件开发的可能。

我不需要假装自己仍然记得所有语法。

我可以重新学习，也可以借助 AI，慢慢恢复曾经拥有的能力。

AI 对我来说，不只是一个提高效率的工具。

它更像是一座桥。

一边连接着我过去做过的软件开发，另一边连接着我现在想要建设的系统。

在不同阶段，我接触过跨境电商、国际贸易、供应链、产品采购、网站建设、内容制作、品牌设计和数据分析。

我研究过不同的平台，包括：

- WordPress

# 四、义乌：从想法走向真实产品

义乌对我来说，是一个非常特别的地方。

在那里，很多抽象的商业概念都会变成具体的东西。

链条、耳钩、吊坠、珍珠、宝石、包装盒、展示道具、卡片、五金件、辅料、工具，每一种产品都可以被看见、触摸和比较。

在电脑上研究产品时，一个设计可以看起来非常完美。

但当真正拿到材料以后，就会发现很多现实问题：

- 尺寸是否合适
- 重量是否能够接受
- 配件是否容易脱落
- 镀层是否耐用
- 珍珠孔位是否正确
- 结构是否能够稳定连接
- 工厂是否愿意小批量生产
- 成本是否能够支撑最终售价
- 图片效果与实物是否一致

这些问题无法只靠想象解决。

必须通过实际购买、测试、组合和修改，一点点得到答案。

我开始接触巴洛克珍珠、月光石、海蓝宝、紫水晶、黄水晶、白蝶贝和各种首饰配件。

我尝试不同的结构：

- 珍珠与金属配件结合
- 天然石与动物造型结合
- 可替换吊坠
- 珍珠颜色变化
- 不同形状的异形巴洛克珍珠
- 项链、耳饰和胸针的多种用途
- 手工金线与天然材料结合

很多方案在图片里很好看，但真正制作时并不一定可行。

有的金线太短，实际操作困难。

有的配件太重，不适合作为耳饰。

有的珠子尺寸稍微大一点，整体比例就会失衡。

有的结构看起来能够固定，真实使用时却可能松动。

每一次调整都让我更明白：

> 产品不是一个漂亮的想法，而是一系列现实条件之间的平衡。

# 五、SIWEI 珠宝项目

我希望 SIWEI 的珠宝不只是普通的组合饰品。

我想尝试把天然材料本身的不规则性保留下来。

巴洛克珍珠的形状并不标准。

每一颗珍珠的纹理、光泽、凹凸和轮廓都不完全相同。

传统工业产品追求统一，而巴洛克珍珠的价值恰恰来自不统一。

这种不规则让我产生了一种共鸣。

人也不是标准化的产品。

我们都有自己的痕迹、限制、缺口和不完美。

但不完美并不等于没有价值。

有时候，真正让一个东西被记住的，恰恰是它无法被复制的部分。

因此，我希望围绕巴洛克珍珠、天然宝石和手工改造，逐渐建立属于 SIWEI 的产品语言。

我尝试过：

- 巴洛克珍珠与金属配件组合
- 珍珠与天然石的颜色搭配
- 异形珍珠替换设计
- 手工金线嵌入
- 珍珠表面开槽
- UV 胶与金属线固定
- 锆石点缀
- 项链、耳饰和胸针结构
- 半定制与随机搭配
- 为每颗天然石设计独立说明卡片

我希望客户收到的不只是一件商品。

也希望他们能够看到材料本身的差异，理解每一颗石头和珍珠都不是完全相同的。

也许未来，客户会收到一张小卡片，上面写着这颗天然材料的特征和一句属于它的话。

这样，一件产品就不仅仅是库存中的一个编号。

它会拥有自己的描述和故事。

# 六、景德镇与陶瓷创作

除了首饰，我也在接触和探索陶瓷。

陶瓷和首饰有很多不同。

首饰往往依赖精确的结构、尺寸和连接关系，而陶瓷则同时包含材料、手感、火、温度、釉料和不可完全控制的变化。

在烧制之前，没有人能够百分之百确定最终结果。

颜色可能发生变化，釉面可能流动，线条可能因为温度和材料产生新的效果。

这种不可预测性让我觉得陶瓷很有吸引力。

它不是单纯把设计复制到器物上，而是设计、材料和火共同完成作品。

我开始研究：

- 高白瓷泥
- 黑料青花
- 高温颜色釉
- 中温釉下彩
- 化妆土
- 哑光釉
- 蓝色系釉面
- 杯型与器型
- 瓷板画
- 手绘图案
- 烧制前后的颜色变化

我尤其喜欢一些反复出现的元素：

- 蝴蝶
- 树
- 月亮
- 梦境
- 夜色
- 蓝色
- 金色
- 黑色
- 红色
- 青花线条

“庄生梦蝶”是我长期想继续发展的主题之一。

蝴蝶可以是梦，也可以是变化。

树代表时间、生长和停留。

月亮则像是一个安静的旁观者，照亮那些无法被完全解释的部分。

我希望把这些元素逐渐形成一个具有识别度的系列，而不是只制作一件孤立的作品。

陶瓷让我重新理解了“控制”这件事。

我们可以认真准备泥、釉、构图和温度，但最终仍然需要接受火带来的变化。

人生也是如此。

我们可以选择过程，却无法控制所有结果。

随着 AI 辅助开发逐渐成熟，我重新开始建立自己的技术体系。

我目前使用或研究的工具包括：

- Codex
- VS Code

- AnythingLLM

- Hostinger VPS
- 数据库

过去，开发一个完整系统意味着需要独立掌握前端、后端、数据库、服务器、部署、测试和维护。

现在，AI 可以帮助完成其中的大量工作。

但我也逐渐认识到，AI 并不会自动让一个项目变好。

如果目标不清晰，AI 会生成大量无关内容。

如果项目边界不明确，AI 会不断增加不需要的功能。

如果没有固定文档，AI 每一次开发都可能采用不同的思路。

因此，我开始采用“文档优先”的方式开发。

在写代码之前，先固定：

- 项目目标
- 功能范围
- 不做什么
- 数据字段
- 页面结构
- 数据库关系
- 分析标准
- 文件目录
- 开发规则
- 部署方式

我希望每个项目都有属于自己的说明文件。

Codex 在开发之前，必须先读取这些文件，然后按照已经确定的规则执行。

这种方式不仅是为了约束 AI，也是为了约束我自己。

当我看到新的想法时，我很容易希望把它加入当前项目。

但一个系统只有在边界清晰时，才有可能真正完成。

# 九、Alibaba.com 国际站 RFQ 项目

RFQ 项目是我目前正在建设的一个重要系统。

它只服务于 Alibaba.com 国际站 RFQ。

它不与 Etsy、Shopify、Reddit、社交媒体或其他业务系统混合。

这个项目的目标不是自动代替我进行报价，也不是抓取大量公开数据。

它首先是一个私人使用的数据整理、分析和长期归档系统。

在 Alibaba.com 国际站中，RFQ 可能以不同方式出现：

- 文字
- 截图
- 图片
- 不同语言
- 产品要求
- 数量信息
- 目标价格
- 包装要求
- 认证要求
- 运输要求
- 客户信息
- 报价记录

这些内容如果只依赖平台页面或人工记忆，很容易随着时间丢失。

因此，我希望建立一个属于自己的系统，把每一条重要 RFQ 保存下来。

系统将遵循一个核心原则：

> 原始信息必须原封不动保存。

如果输入的是西班牙语，就保留西班牙语原文。

如果输入的是截图，就保存原始截图。

如果 AI 提取了结构化字段，也不能覆盖原始内容。

如果系统进行了翻译，翻译结果必须与原文分开保存。

如果系统生成了分析结论，分析结果也必须与事实字段分开保存。

每条 RFQ 可以包含三层数据：

## 原始数据

- 原始文字
- 原始语言
- 原始截图
- 原始图片
- 原始附件
- 原始链接
- 录入时间

## 结构化数据

- RFQ 标题
- 产品名称
- 产品类目
- 国家或地区
- 采购数量
- 单位
- 目标价格
- 币种
- 材质
- 尺寸
- 颜色
- 包装要求
- 认证要求
- 运输要求
- 定制要求
- 发布时间
- 截止时间

## 分析结果

- 信息完整度
- 需求明确程度
- 是否适合报价
- 潜在风险
- 缺失信息
- 建议确认的问题
- 建议报价策略
- 客户意向判断
- 后续跟进建议

如果报价后能够看到客户信息，系统还可以保存：

- 客户姓名
- 公司名称
- 国家
- 邮箱
- 电话
- WhatsApp
- 公司网站
- Alibaba.com 账户信息
- 人工备注
- 关联的历史 RFQ

GitHub 仓库只保存系统源代码、文档、字段、分析规则和数据库设计。

真实 RFQ 内容、截图、报价记录和客户信息，不会保存到 GitHub。

实际数据将长期保存在部署于 Hostinger 的私人网站和数据库中。

对我来说，这个系统的价值不只是提高一次报价的效率。

更重要的是，它能够把零散的业务经验逐渐沉淀为长期数据。

随着记录不断增加，我可以逐渐分析：

- 哪些国家的 RFQ 更值得关注
- 哪些产品需求更真实
- 哪些采购数量更容易成交
- 哪些报价策略更有效
- 哪些客户值得长期跟进
- 哪些 RFQ 存在明显风险
- 自己为什么错过某些机会
- 自己在哪些地方判断错误

经验如果只存在于记忆中，很容易随着时间消失。

当经验被记录下来，它才有机会变成可以重复使用的知识。

# 十、我的本地知识库

随着我保存的资料越来越多，我开始建立自己的本地知识库。

我收集和整理过：

- Etsy 官方规则
- Seller Handbook
- 平台政策
- 商品页面
- Listing 图片
- Listing HTML
- 店铺信息
- 产品资料
- 分析报告
- 项目开发文档

我使用 Chroma 建立本地向量数据库，希望让 AI 能够检索这些内容，而不是每次都从零开始回答。

这个系统的目的不是简单存放文件。

我希望每份资料都能够被整理、拆分、标注和分析。

例如，一份平台规则 PDF，可以生成：

- 原始文件
- 元数据
- 内容摘要
- 关键规则
- 风险提示
- 适用范围
- 引用位置
- AI 分析结果

一条商品 Listing，可以保存：

- 原始图片
- 原始 HTML
- 商品信息
- 图片分析
- 页面分析
- 评论分析
- SEO 分析
- 改进建议

知识库让我逐渐意识到：

> 资料只有被整理和检索，才能真正成为知识。

单纯下载很多文件，并不等于掌握了这些信息。

# 十一、网站与长期保存

我希望自己的项目不只存在于聊天记录里。

聊天记录很方便，但它不是稳定的项目管理方式。

一个重要的决定，如果只存在于一段对话中，几个月后可能很难找到。

因此，我开始把重要内容保存到：

- GitHub
- Markdown 文档
- 本地目录
- 数据库
- 网站
- 知识库
- 图片和设计文件

我使用 Hostinger VPS 建设网站，也尝试过 WordPress、Shopify 和独立系统。

网站对我来说不只是展示产品的页面。

它也可以成为一个长期保存信息的地方。

代码可以保存在 GitHub。

真实业务数据可以保存在私人数据库。

产品图片可以保存在独立存储中。

项目规则可以写进文档。

这样，即使以后更换工具、模型或开发方式，重要内容仍然能够保留下来。

# 十二、我经历过的犹豫和改变

我的道路并不稳定。

我也不是从一开始就知道自己最终要做什么。

我曾经在多个方向之间来回思考：

- 做珠宝还是做陶瓷
- 做零售还是做批发
- 做 Etsy 还是做独立站
- 做 Alibaba.com 国际站还是做其他渠道
- 做产品还是做软件
- 做内容还是做供应链
- 做现有业务还是研究新技术

有时，方向变化并不是因为之前的决定完全错误。

而是因为我在实践中获得了新的信息。

真正做过以后，才会发现哪些想法可行，哪些成本太高，哪些市场并不适合自己，哪些项目需要暂时停止。

我曾经也会因为改变方向而怀疑自己。

好像一旦更换路线，就意味着过去的努力全部浪费。

后来我逐渐接受：

> 改变方向不一定是失败，也可能是逐渐看清现实的过程。

过去做过的事情不会完全消失。

珠宝项目让我理解产品、材料和供应链。

陶瓷项目让我理解手工制作、审美和不可控性。

跨境业务让我理解平台、客户和市场。

网站开发让我理解内容、结构和长期积累。

AI 开发让我重新获得建设系统的能力。

这些经历最终会在某个地方连接起来。

# 十三、我对失败的理解

我并不认为失败是一件浪漫的事情。

失败会消耗时间、金钱和信心。

有些失败不会立即带来经验，只会让人感到疲惫。

有些项目停止以后，也不一定能够立刻总结出清晰的原因。

因此，我不想简单地说“失败是成功之母”。

我更愿意相信：

> 失败只有在被记录、被分析并改变下一次行动时，才可能产生价值。

如果同样的错误不断重复，那么失败就只是重复付出的代价。

这也是我为什么越来越重视数据、文档和复盘。

我希望记录：

- 当时为什么做出这个决定
- 使用了哪些信息
- 预期结果是什么
- 实际发生了什么
- 哪些判断是正确的
- 哪些判断出现偏差
- 下一次应该怎样调整

我无法保证自己不再失败。

但我希望自己能够减少毫无意义的重复。

# 十四、关于时间

时间对每个人都公平，也不公平。

每个人的一天都是二十四小时。

但每个人真正能够自由使用的时间并不相同。

有人需要照顾家庭，有人需要应对身体问题，有人需要承受经济压力，有人需要在工作之后才能开始自己的项目。

我很早就意识到，时间不是一种可以无限储存的资源。

很多事情不能一直等到“以后有条件了再做”。

因为所谓的以后，并不一定会按照我们的想象到来。

但这种认识也曾经让我着急。

我想同时做很多事情，害怕放弃任何可能性，害怕时间不够。

结果反而容易让精力变得分散。

现在我正在学习另一种方式：

不是同时做完所有事情，而是认真完成当前最重要的一件事。

完成一个项目，再进入下一个阶段。

把大目标拆成小步骤。

今天写清楚一个文档。

明天固定一个数据字段。

后天完成一个页面。

一个系统可能需要很长时间，但它至少会每天多存在一点。

# 十五、关于身体与不确定性

我的一些人生选择，与个人身体状况带来的长期感受有关。

我不打算在这里公开具体的医学信息。

这不是因为我否认这部分经历，而是因为一个人的身份不应该只由某一种身体状况定义。

我只是比很多人更早地知道，平静并不是理所当然的。

有些风险可能突然发生。

有些计划可能被迫中断。

有些普通人不需要反复考虑的事情，对另一些人来说可能需要提前做好准备。

这种经历曾经让我害怕。

但它也让我不断思考：

如果人生存在无法控制的部分，那么我还能控制什么？

我可以控制今天是否开始。

我可以控制是否把一个想法写下来。

我可以控制是否认真完成一件产品。

我可以控制是否保存自己的经验。

我可以控制是否在有能力的时候，尽量帮助别人。

我可以控制自己选择怎样面对不确定性。

我无法决定所有结果。

但我仍然可以决定中间的路线。

# 十六、我为什么想留下这些记录

互联网充满了成功故事。

很多故事在回头叙述时，都显得非常清晰。

仿佛主人公从一开始就知道目标，也知道每一步应该怎么走。

但真实的人生往往不是这样。

真实的人生会犹豫，会绕路，会做出错误判断，也会在很长时间里看不到明显结果。

一个人的一生，除了最终取得的结果，也应该包括他曾经怎样思考、怎样选择以及怎样面对自己的处境。

# 十九、我的开发原则

## 1. 先固定目标，再开始开发

不要在目标模糊时让代码快速增长。

## 2. 一个项目只解决一类问题

RFQ 系统不加入 Etsy、Shopify、Reddit 和其他无关功能。

## 3. 原始数据必须保留

翻译、提取和分析都不能覆盖原始内容。

## 4. 文档优先

重要规则不能只存在于记忆或聊天记录中。

## 5. 不做无谓升级

新功能必须真正服务于当前目标。

## 6. 小步完成

先完成能够使用的第一版，再考虑扩展。

## 7. 真实数据与代码分离

GitHub 保存代码和规则，不保存客户隐私及真实业务数据。

## 8. 保持可维护性

未来的我，应该能够看懂今天写下的项目。

# 二十、我的创业原则

## 不盲目追求规模

规模并不等于价值。

在没有稳定流程之前，快速扩大只会放大问题。

## 不因为别人成功就复制所有路径

别人的资源、条件和目标可能与我不同。

参考可以，照搬未必适合。

## 产品必须经过真实测试

图片好看不代表实物可用。

设计必须接受材料、结构、重量、成本和工艺的检验。

## 数据比模糊印象更可靠

尽可能记录曝光、点击、询盘、报价、回复、成本和结果。

## 不隐藏失败

失败的项目也应该留下原因和结论。

## 尊重长期积累

一个品牌、一个系统和一种设计语言，都不可能只靠几天完成。

# IDEAS

## 目录用途

记录尚未成熟的想法、功能设想、产品构思和未来可能性。写在这里不代表已经决定执行。

## 适合记录的内容

- 新产品或新功能的初步构思
- 尚待验证的商业机会
- 对现有项目的改进设想
- 未来可能尝试的方向

## 不适合记录的内容

- 已经正式启动的项目说明
- 已确认的执行计划或重要决定
- 没有任何说明的单句备忘

## 推荐的文件命名方式

使用 `YYYY-MM-DD-想法主题.md`；不需要日期时可使用 `想法主题.md`。

## 简单示例

`2026-07-15-可更换珠宝组件.md`：记录产品构思、可能的使用场景以及仍需验证的问题。
